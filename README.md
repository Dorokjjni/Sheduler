# Планировщик — PWA с поясняющими комментариями

Устанавливаемое веб-приложение с боковым меню и четырьмя независимыми разделами:

- Календарь;
- Регулярное;
- Планирование;
- Список задач и целей.

На текущем этапе каждый раздел использует одинаковый интерфейс списка задач, но данные разделов хранятся отдельно в `localStorage`.

## Где находятся комментарии

- `index.html` — комментарии объясняют структуру страницы и назначение блоков;
- `styles.css` — комментарии объясняют каждый логический раздел оформления;
- `app.js` — подробно описаны данные, функции и последовательность действий;
- `service-worker.js` — объяснены установка, кеширование, активация и офлайн-режим;
- `manifest-explained.jsonc` — учебная копия manifest с комментариями;
- `CODE-GUIDE.md` — общая карта проекта и порядок работы приложения.

Файл `manifest.webmanifest` оставлен без комментариев, потому что это строгий JSON: обычные комментарии `//` и `/* */` делают его недействительным. PNG-иконки являются бинарными изображениями, поэтому комментарии нельзя записать внутрь них. Их назначение описано в `CODE-GUIDE.md`.

## Возможности

- работает на iPhone, Android и компьютере;
- устанавливается на домашний экран;
- работает без интернета после первого открытия;
- сохраняет задачи отдельно для каждого раздела;
- переносит старые задачи в раздел «Список задач и целей»;
- адаптировано под безопасные зоны iPhone.

## Локальный запуск на Windows

Service Worker не работает при открытии файла напрямую через `file://`. Запустите локальный сервер.

```bat
cd путь\к\папке\Sheduler
py -m http.server 8080
```

Откройте в браузере:

```text
http://localhost:8080
```

## Публикация на GitHub Pages

Загрузите новые версии файлов в корень репозитория и создайте коммит. GitHub Pages обновится после публикации изменений в ветку `main`.

Пример сообщения коммита:

```text
Добавлены подробные комментарии к коду
```
