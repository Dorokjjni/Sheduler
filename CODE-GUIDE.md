# Карта кода проекта

## 1. Что происходит при открытии приложения

1. Браузер загружает `index.html`.
2. `index.html` подключает `styles.css`, `app.js` и `manifest.webmanifest`.
3. `app.js` находит элементы страницы через `document.querySelector`.
4. Из `localStorage` загружаются задачи и последний выбранный раздел.
5. Функция `renderSchedule()` меняет заголовок и подсвечивает выбранный пункт меню.
6. Функция `renderTasks()` создаёт видимые элементы задач.
7. После полной загрузки страницы регистрируется `service-worker.js`.
8. Service Worker сохраняет основные файлы в кеше для офлайн-запуска.

## 2. Назначение файлов

### `index.html`

Содержит структуру интерфейса:

- боковое меню;
- кнопку с тремя полосками;
- заголовок текущего раздела;
- форму добавления задачи;
- список задач;
- шаблон одной задачи;
- подсказку установки PWA.

### `styles.css`

Отвечает за внешний вид:

- цвета и шрифты;
- расположение элементов;
- выдвижное меню;
- оформление задач и checkbox;
- адаптацию под iPhone;
- анимации и доступность.

### `app.js`

Отвечает за поведение:

- открытие и закрытие меню;
- переключение разделов;
- добавление, выполнение и удаление задач;
- сохранение в `localStorage`;
- установку PWA;
- отображение состояния сети;
- регистрацию Service Worker.

### `manifest.webmanifest`

Содержит настройки установки PWA:

- название приложения;
- короткое название;
- стартовый адрес;
- режим отдельного окна `standalone`;
- цвета;
- ориентацию экрана;
- список иконок.

Это строгий JSON, поэтому комментарии внутри него запрещены. Учебная версия с комментариями находится в `manifest-explained.jsonc`; она не подключена к приложению.

### `service-worker.js`

Создаёт офлайн-режим:

- при установке кеширует основные файлы;
- при обновлении удаляет старые кеши;
- для HTML сначала пробует сеть, затем кеш;
- для остальных файлов сначала проверяет кеш.

### Папка `icons`

- `icon-180.png` — иконка домашнего экрана iPhone;
- `icon-192.png` — стандартная небольшая PWA-иконка;
- `icon-512.png` — большая PWA-иконка;
- `icon-maskable-512.png` — иконка с безопасными отступами для разных форм значков.

## 3. Где хранятся задачи

Задачи находятся только в браузере пользователя в `localStorage` под ключом:

```text
pwa-scheduler.schedules.v2
```

Структура данных примерно такая:

```json
{
  "calendar": [],
  "regular": [],
  "planning": [],
  "task-list": []
}
```

Каждый массив содержит объекты задач с полями `id`, `text`, `completed` и `createdAt`.

## 4. Как добавить новый раздел

Нужно изменить два места:

1. Добавить кнопку в `<nav class="schedule-navigation">` файла `index.html`.
2. Добавить такой же ключ и название в объект `SCHEDULES` файла `app.js`.

Значение `data-schedule` в HTML должно точно совпадать с ключом в JavaScript.

## 5. Как безопасно обновлять Service Worker

После важного изменения файлов увеличьте версию:

```js
const CACHE_NAME = "pwa-scheduler-v4";
```

Это помогает браузеру удалить старый кеш и загрузить новую версию приложения.
